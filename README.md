# BizPpurio AI Optimizer

> 다우기술 비즈뿌리오를 위한 AI 캠페인 최적화 프로토타입

기업 메시지 캠페인의 **발송 시간**, **타겟 세그먼트**, **성과 예측**을 머신러닝으로 최적화하는 데모 프로젝트입니다. 다우기술 AX(AI Transformation) 직무 지원을 위한 포트폴리오로 제작되었습니다.

---

## 📌 프로젝트 동기

### 비즈뿌리오, 그리고 AI의 다음 단계

비즈뿌리오는 다우기술 별도 매출의 약 절반을 차지하는 핵심 사업이며, 19년 연속 SMS 시장점유율 1위를 지켜온 강력한 자산입니다. 현재 AI 적용은 사내 **XI Lab**을 중심으로 활발히 진행되고 있으며, 메시지 문구 자동 생성 등 생성형 AI 기능이 도입되어 있습니다.

본 프로토타입은 그다음 단계, 즉 **데이터 기반 의사결정 레이어**를 비즈뿌리오에 더하는 시나리오를 제안합니다.

| 영역 | 현재 비즈뿌리오 | 본 프로젝트 제안 |
|---|---|---|
| 메시지 문구 생성 | ✅ 생성형 AI 적용 중 | — |
| 발송 시간 최적화 | — | ✅ 시간대별 전환율 예측 |
| 타겟 세그먼트 추천 | — | ✅ 캠페인 목적별 추천 |
| 성과 예측 | — | ✅ 발송 전 예상 KPI 제공 |

매출 기여도가 가장 큰 서비스에 예측·추천 레이어를 더하는 것이 다우기술의 AI 전환에서 가장 임팩트가 큰 진입점이라는 가설에서 출발했습니다.

## 🏗 시스템 아키텍처

3-레이어 구조로 설계되어 있으며, 각 레이어는 독립 모듈로 분리되어 단계별 검증과 교체가 가능합니다.

```
┌────────────────────────────────────────────────┐
│  Service Layer  (FastAPI + React)              │
│  └─ 캠페인 조건 입력 → 추천·예측 결과 시각화   │
└──────────────────────▲─────────────────────────┘
                       │
┌──────────────────────┴─────────────────────────┐
│  ML Layer  (XGBoost / LightGBM)                │
│  ├─ 발송 시간 예측 모델                        │
│  ├─ 캠페인 전환율 예측 모델                    │
│  └─ 타겟 세그먼트 추천 모델                    │
└──────────────────────▲─────────────────────────┘
                       │
┌──────────────────────┴─────────────────────────┐
│  Data Layer  (합성 데이터 Generator)           │
│  └─ 업종·시간·요일·세그먼트별 현실 패턴 반영   │
└────────────────────────────────────────────────┘
```

## 📁 디렉토리 구조

```
DAOU-AX-Prototype/
├── backend/                  # Python: 데이터 + ML + API
│   ├── src/
│   │   ├── data/             # 합성 데이터 생성 (Layer 1)
│   │   ├── ml/               # 학습·추론 모델 (Layer 2)
│   │   └── api/              # FastAPI 엔드포인트 (Layer 3)
│   ├── scripts/              # 데이터 생성·모델 학습 진입점
│   ├── tests/                # 스모크 테스트
│   └── models/               # 학습된 모델 산출물
├── frontend/                 # React + Vite + TypeScript
├── data/                     # 생성된 합성 데이터
├── notebooks/                # EDA · 모델 검증
└── docs/                     # 아키텍처 · 데이터 · API 문서
```

## 📊 데이터에 관하여

본 프로젝트는 **합성 데이터**를 사용합니다. 실제 비즈뿌리오 운영 데이터에 접근할 수 없으므로, 업종별·시간대별 메시지 캠페인 전환율의 일반적 패턴을 가정하여 데이터를 생성합니다. 자세한 패턴 정의와 가정은 추후 [docs/data_schema.md](docs/data_schema.md)에 정리됩니다.

본 프로젝트의 목적은 **데이터 자체의 정확성보다 "이런 데이터가 있다면 이렇게 가치를 만들 수 있다"는 시스템 설계와 모델링 접근을 보여주는 것**에 있습니다.
